> **v2 변경 이력**: 2026-03-11 외부 리서치 1차/2차 결과를 반영하여 전면 개정. 주요 변경: 포지셔닝 전환("정보 서비스" → "실행 도구"), 전투력 측정기 역할 조정(유입 훅으로 한정), 체크리스트 4축 재구성, 공유 시스템 이원화, 수익화 모델 조정, 검증된 시장 데이터 반영.

# Part 1. 문제정의 (Problem Definition)

---

## 1. 문제 한 줄 요약

> **한국 출산 직후 30일은 아빠가 행정·예산·준비를 병행해야 하는 집중 구간인데, 이를 체계적으로 안내하고 실행시키는 도구가 없다.**

---

## 2. 문제의 맥락

### 2-1. 시장 상황 (리서치 검증 완료)

한국의 아빠 육아 참여는 더 이상 소수 사례가 아니다. 고용노동부 기준 2024년 육아휴직자 132,535명 중 남성은 41,820명(31.6%)이었고, 2025년 1~9월에는 육아휴직급여 수급자 중 남성 비율이 36.8%까지 상승했다. 배우자 출산휴가는 2025년 기준 20일로 확대되었고, 사용률은 55.9%에 달한다.

출산 모수도 최근 반등 신호가 있다. 2024년 출생아 수는 238,343명으로 전년 대비 3.6% 증가했고, 2025년 1월은 전년동월 대비 11.6% 증가했다.

산후조리원은 사실상 기본 인프라다. 보건복지부 2024년 산후조리 실태조사 기준 이용률 85.5%, 평균 비용 286.5만 원, 평균 이용기간 12.6일이다.

그러나 **제도와 인식이 빠르게 변하는 것에 비해, 아빠를 위한 실행 도구는 거의 없다.** 육아 앱(베이비빌리, 베이비노트 등)은 이미 엄마→아빠 공유를 전제로 설계되어 있지만, 이들은 "기록 도구"이지 "실행 도구"가 아니다. 아기의 수유·수면을 기록하는 데 집중하며, 아빠에게 "지금 뭘 해야 하는지"를 알려주지 않는다.

### 2-2. 출산 직후 30일: 아빠의 행정 집중 구간

한국 출산 직후 30일은 아빠가 실제 실행 주체가 되기 쉬운 구간이다:

**D+0~D+3 (출산 직후)**: 아내는 분만 회복 중이고, 아빠가 산후조리원 입소를 주도한다. 조리원에 가져갈 것(빨대, 멀티탭, 가습기 등)을 챙기고, 가족에게 연락하고, 첫 면회를 한다.

**D+3~D+14 (조리원 기간)**: 아빠의 행정 폭탄 구간이다. 온라인 출생신고(정부24), 행복출산 원스톱 서비스(건강보험·아동수당·부모급여 일괄 신청), 첫만남이용권(첫째 200만 원, 둘째 이상 300만 원, 60일 이내 신청), 배우자 출산휴가 신청을 초기 2주 안에 우선 처리하게 된다. 산후도우미 지원사업은 출산예정일 40일 전부터 출산 후 30일까지 신청 가능하므로 이 신청 윈도우도 함께 관리해야 한다. 이 모든 정보가 정부 사이트 여기저기 흩어져 있고, 아무도 순서를 알려주지 않는다.

**D+14~D+30 (퇴소 후 집)**: 아내와 아기가 집에 온다. 새벽 수유 교대, 기저귀 교체, 아기 목욕을 실전으로 시작하면서, 동시에 집 환경 세팅(수유 공간, 기저귀 교체대 배치, 냉장고 채우기)을 해야 한다. 산후도우미가 오면 조율도 필요하다.

이 30일은 아빠 인생에서 가장 많은 "처음"이 동시에 몰려오는 시기인데, 체계적 가이드가 전무하다.

### 2-3. 기존 솔루션의 한계

| 기존 솔루션 | 성격 | 한계 |
|------------|------|------|
| 베이비빌리, 베이비노트 | 기록 도구 (수유·수면 로그) | 아빠에게 "뭘 해라"를 알려주지 않음. 커플 공유는 있지만 아빠 전용 온보딩/가이드 부재 |
| 네이버 검색/맘카페 | 정보 탐색 | 엄마 관점, 정보 파편화, 시의성 낮음 |
| 정부24/행복출산 원스톱 | 행정 도구 | 기능은 있지만 "언제, 어떤 순서로" 하라는 가이드가 없음 |
| 유튜브/블로그 | 콘텐츠 | 체계적이지 않음, 시기별 맞춤 불가 |
| 주변 지인 | 구전 | 운에 의존, 체계적이지 않음 |

핵심 공백: **아빠 전용 정체성 + 시점별 실행 체크리스트 + 행정 타임라인 + 예산 시뮬레이터를 한 번에 묶는 "PM형 실행 도구"가 존재하지 않는다.**

### 2-4. 핵심 문제 구조화

```
[근본 원인]
출산 직후 30일의 아빠 업무가 구조화되지 않았다
  │
[직접 원인 — 도구 부재]
├─ 시점별 할 일을 알려주는 실행 도구 없음
├─ 행정 순서/기한을 정리한 타임라인 없음
├─ 총 비용을 한눈에 보여주는 시뮬레이터 없음
└─ 아빠 관점의 준비물 체크리스트 없음
  │
[증상 — 아빠 측]
├─ 뭘 준비해야 하는지 모름
├─ 행정을 언제 어떤 순서로 해야 하는지 모름
├─ 총 비용 감이 없음
├─ 실전 노하우가 없음 (빨대가 필요한 줄 몰랐음)
└─ 물어볼 곳 없음
  │
[증상 — 엄마 측]
├─ "남편에게 하나하나 알려줘야 한다" (정신적 부담)
├─ 체계적으로 보내줄 수 있는 도구가 없다
└─ 남편에게 보내면 바로 실행하게 되는 '실행 스위치'가 필요하다
```

## 3. 우리의 관점 (Point of View)

### 제품 포지셔닝 (리서치 반영 v2)

> **파파레벨은 아빠를 위한 육아 정보 서비스가 아니라, 엄마가 남편에게 보내는 실행 스위치이자 아빠 되기 프로젝트의 PM 도구다.**

### 아빠에게
> 출산은 프로젝트다. 출생신고, 지원금 신청, 준비물 구매, 조리원 입소 — 전부 기한이 있는 업무다.
> 파파레벨은 이 프로젝트의 타임라인과 체크리스트를 제공한다.

### 엄마에게
> 남편에게 일일이 말하지 마세요. 이거 하나 보내주면 됩니다.
> 엄마의 니즈는 더 많은 정보가 아니라, 남편에게 보내면 바로 움직이게 되는 한 장짜리 실행물이다.

### 핵심 산출물 원칙 (리서치 반영)
> 파파레벨의 핵심 산출물은 읽을거리가 아니라, 카카오톡 링크로 보내기 쉬운 체크리스트와 오늘 해야 할 일 카드다. 이미지/PDF는 보조 포맷으로 둔다.

## 4. 타겟 사용자

### Primary User: 아빠 (실행 주체)
- 1차 타겟: 첫째 아이, 출산 전 1개월 ~ 출산 후 1개월 (D-30 ~ D+30)
- 확장 타겟: 출산 후 6개월까지
- 연령: 28~40세
- 특성: 직장인, 의지는 있으나 시작점과 순서를 모름

### Secondary User: 엄마 (유입 엔진 + 실행 트리거)
- 남편에게 "이거 해봐" / "이거 확인해" 형태로 카톡 전송
- 맘카페/단톡방에서 "우리 남편한테 이거 보냈더니ㅋㅋ" 공유
- 파파레벨의 바이럴은 아빠가 아니라 엄마가 만든다

### Anti-Persona
- 육아에 관심 없는 아빠 (도구를 줘도 안 쓸 사람)
- 이미 베이비빌리 등으로 충분히 관리 중인 적극적 커플

## 6. 성공 기준

이 문제가 "해결되었다"고 볼 수 있는 지표:

- 아빠가 출산 후 30일 동안 "이번 주에 뭘 해야 하는지" 파파레벨에서 바로 확인할 수 있다
- 아빠가 행정 순서(출생신고 → 건강보험 → 지원금)를 타임라인으로 한눈에 파악할 수 있다
- 아빠가 총 비용을 3분 내에 시뮬레이션할 수 있다
- 엄마가 "남편에게 보내기"로 체크리스트나 할 일 카드를 전달할 수 있다
- 전투력 측정기 결과 카드가 자발적으로 공유된다

### Feature 1: 아빠 육아 전투력 측정기 (유입 훅)

#### 개요
5개 질문에 답하면 아빠의 "육아 전투력 등급"을 진단하고, SNS 공유용 결과 카드를 생성한다.

#### 역할 정의 (v2 변경)
- **역할**: 바이럴 유입 훅. 재미와 공감으로 첫 방문을 만든다.
- **역할이 아닌 것**: 리텐션 기능이 아니다. 1회성 체험이 자연스럽다.
- 결과 화면에서 실행 도구(체크리스트, 행정 타임라인 모듈, 예산)로 전환시키는 것이 핵심.

#### 사용자 플로우

```
[랜딩] "당신의 육아 전투력은?"
  │
  ▼
[질문 1~5] 객관식 선택 (탭/클릭)
  │
  ▼
[결과 화면]
  ├─ 등급 카드 (이미지)
  ├─ 등급 설명 + 재치 있는 코멘트
  ├─ [카카오톡 공유] [인스타 스토리 공유] [이미지 저장]
  │
  ├─ [CTA 1] "이번 주 아빠가 해야 할 3가지 →"        ← v2 신규
  ├─ [CTA 2] "행정 타임라인 바로 보기 →"             ← 체크리스트 내 모듈 딥링크
  └─ [CTA 3] "첫 아이, 얼마나 들까요? →"
```

#### 질문 설계

| # | 질문 | 보기 | 배점 기준 |
|---|------|------|----------|
| 1 | 아기 울음소리에 몇 초 만에 눈을 뜨나요? | 3초 이내 / 10초 / 아내가 깨워줌 / 모름(아직 미출산) | 경험도 + 반응속도 |
| 2 | 기저귀 교체, 자신 있나요? | 눈 감고도 가능 / 해봤다 / 이론만 / 기저귀 종류를 모름 | 실전 경험 |
| 3 | 분유 적정 온도를 어떻게 확인하나요? | 손목 테스트 / 입으로 / 온도계 / 분유를 타본 적 없음 | 실무 지식 |
| 4 | 아기와 단둘이 외출한 최장 시간은? | 3시간+ / 1시간 / 편의점까지 / 아직 없음(미출산 포함) | 독립 돌봄 능력 |
| 5 | 아기 재우기, 본인만의 비법이 있나요? | 3가지 이상 / 하나 있음 / 매번 다름 / 재운 적 없음 | 노하우 축적도 |

#### 미출산 사용자 보정 규칙 (v2 신규)

- `아직 미출산` 또는 이에 준하는 응답이 3개 이상이면 `예비 아빠 모드`로 분기한다.
- 예비 아빠 모드에서는 낮은 점수로 직접 낙인찍지 않고, `지금 준비하면 되는 아빠` 톤의 결과 카드와 카피를 사용한다.
- 이 경우 핵심 CTA는 `출산 전 체크리스트`, `행정 준비`, `예산 시뮬레이터`로 고정하고, 퍼센타일 비교 문구는 노출하지 않는다.

#### 등급 체계

| 등급 | 점수 | 칭호 | 결과 코멘트 |
|------|------|------|------------|
| S | 23~25 | 전설의 슈퍼대디 | "아내가 이 결과를 보면 의심할 수 있습니다" |
| A | 18~22 | 숙련된 육아전사 | "장모님이 인정한 사위 레벨입니다" |
| B | 13~17 | 성장형 아빠 | "의지는 S급, 스킬은 업데이트 중" |
| C | 8~12 | 루키 아빠 | "모든 레전드도 여기서 시작했습니다" |
| D | 5~7 | 관전 아빠 | "지금 이 테스트를 누가 보내줬는지 생각해보세요" |

- 모든 등급의 코멘트가 재미있어야 공유된다 (낮은 등급일수록 더 웃기게)
- S/A/B/C/D 등급은 `출산 경험 응답`이 충분한 사용자에게만 기본 적용한다.

### Feature 2: D-30~D+30 실행 체크리스트 (핵심 가치)

#### 개요 (v2 전면 재설계)
출산 전후 30일을 4축으로 나누어, 시점별로 "지금 뭘 해야 하는지"를 알려주는 실행 도구. 단순 정보 나열이 아니라, 각 항목에 "왜 / 언제 / 누가 / 카톡으로 보내기"를 묶는다.

#### 핵심 목적
- **MVP의 핵심 잔존 가치** (리서치 반영: 유입은 전투력 측정기, 잔존은 여기)
- 읽는 정보가 아닌 실행 도구
- 엄마가 아빠에게 바로 보낼 수 있는 실행 카드
- 제휴 마케팅 수익화 (항목별 구매 링크)
- 단건 유료 실행 팩으로의 업셀

#### 4축 구조

```
[시작] 출산 예정일 또는 출산일 입력
  │

  │
  ├─ 축 1: 출산 전 준비 (D-30 ~ D-Day)
  ├─ 축 2: 산후조리원 (D-Day ~ D+14)
```

>
> **지역/기준일 안내 원칙**: 지역별 차이가 있는 항목은 `지역에 따라 달라질 수 있습니다. 아래 내용은 일반적인 기준이며, 거주지 기준으로 다시 확인하세요.` 안내문구를 먼저 보여주고, `기준일`과 `최종 검수일`을 함께 노출한다.

#### 사회적 증거 운영 원칙
- MVP에서는 `아빠 N%가 체크`, `상위 X%`, `같은 지역 평균과 비교` 같은 비교형 문구를 노출하지 않는다.
- 관련 데이터가 필요하면 내부 익명 집계만 유지하고, 대외 노출은 최소 표본 기준이 정해진 뒤 재검토한다.

#### 기술 요구사항
- 출산(예정)일 입력 → D-Day 기준 자동 시점 계산
- 체크 상태 저장 (localStorage, 추후 계정 연동)
- 익명 사용량 집계 API (내부 분석용)
- 항목별 쿠팡파트너스 링크 관리
- 이미지 생성 (체크리스트 요약 카드)

### Feature 3: 출산 예산 시뮬레이터

#### 개요

- 사회적 증거(`아빠 N%가 체크`, `상위 X%`, `같은 지역 평균 비교`)는 MVP에서 보류한다.

**작업 가설**

- 체크리스트, 행정 타임라인, `이번 주 아빠 할 일`, 예산 기본값은 `출산 예정/출산 완료`, `출산 예정일 또는 출산일`, `거주 지역`, `맞벌이 여부`를 공용 입력값으로 우선 사용한다.
- MVP는 로그인 없이 `localStorage + 익명 집계`로 시작하고, 공유 링크는 개인 상태가 아니라 콘텐츠 컨텍스트만 전달한다.
- 공유 이미지는 `OG/고정 프리뷰는 서버 생성`, `개인화 결과 카드/영수증은 클라이언트 생성`으로 나눈다.

**미해결 질문**

- 없음. 위 항목은 2026-03-12 결정사항을 기준으로 고정한다.

## 6. 기술 스택

| 영역 | 기술 | 이유 |
|------|------|------|
| 프레임워크 | Next.js (App Router) | SSR/SSG, OG 이미지 생성, CK 경험 있음 |

| DB | Supabase (PostgreSQL) | 무료 티어, 집계 데이터 |
| 분석 | GA4 (MVP 필수), Mixpanel (옵션) | MVP 계측 단순화 + 후속 퍼널 확장 |
| 공유 | Kakao SDK | 카톡 공유 필수 |
| 제휴 | 쿠팡파트너스 API | 구매 링크 + 수익 |

### 6-1. 공용 데이터 계약 원칙

- 구조화 데이터는 `ChecklistItem`, `PolicyBenefit`, `BudgetItem`, `QuizQuestion`, `ShareTemplate`, `EventSpec` 계약으로 먼저 고정한다.
- 정책/행정 데이터는 최소 `source`, `effective_date`, `verified_at`, `region_scope` 메타필드를 가진다.
- 기한 기반 항목은 `due_offset_days` 또는 동등한 상대일 규칙을 포함해 D-Day 계산 오류를 줄인다.
- 서버 저장이 필요한 동적 데이터는 익명 집계와 운영 메타데이터로 제한한다.

| 행정/지원금 완벽 가이드 | 4,900원 | 신청 순서 + 스크린샷 가이드 + 기한 캘린더 |
| 조리원 아빠 서바이벌 키트 | 4,900원 | 조리원 필수템 + 아빠 행동 매뉴얼 + 아내 멘탈케어 가이드 |

> **포맷 결정**: 엄마가 남편에게 보내는 기본 포맷은 `카카오톡 링크`로 고정한다. PDF는 유료 실행 팩 또는 보조 포맷으로 후속 검토한다.

#### 수익화 검증 퍼널 (v2 신규)

| 베이비빌리 등이 아빠 기능 강화 | 차별화 약화 | 중간 | CK 실전 경험 기반 콘텐츠(빨대, 멀티탭 등)는 대기업 앱이 만들 수 없음. 속도로 승부 |
| 정부 지원금 정책 변경 | 시뮬레이터 정확도 하락 | 높음 | 분기별 업데이트 + 연도 명시. 행정 타임라인은 순서 자체는 크게 안 변함 |
| 쿠팡파트너스 수수료 변경 | 제휴 수익 감소 | 중간 | 다채널 제휴 |
| 출산율 지속 하락 | 시장 축소 | 높음 | 객단가 향상 + 글로벌 확장 (PapaLevel 영어) |
| 유료 실행 팩 전환이 안 됨 | 수익화 실패 | 중간 | 제휴 수익으로 보전 + 포맷/가격 A/B 테스트 |
